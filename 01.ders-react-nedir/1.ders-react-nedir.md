# Ders 1: React Nedir?

## React Nedir?

React, kullanıcı arayüzleri (UI) oluşturmak için kullanılan açık kaynaklı bir **JavaScript kütüphanesidir**. Facebook (şimdi Meta) tarafından geliştirilmiş ve 2013 yılında açık kaynak olarak yayınlanmıştır.

React bir **framework değil, kütüphanedir**. Bu, sadece UI katmanına odaklandığı anlamına gelir. Routing, state management gibi diğer ihtiyaçlar için ek kütüphaneler kullanılır.

---

---

## Neden React?

1. **Component Tabanlı Mimari**: UI'ı küçük, yeniden kullanılabilir parçalara böler
2. **Virtual DOM**: Gerçek DOM'u verimli şekilde günceller, performans artışı sağlar
3. **Tek Yönlü Veri Akışı**: Veri akışını takip etmek ve debug yapmak kolaydır
4. **Büyük Ekosistem**: Binlerce hazır kütüphane ve araç
5. **Güçlü Topluluk**: Aktif geliştirici topluluğu ve bol kaynak
6. **İş Fırsatları**: En çok aranan frontend teknolojilerinden biri

---

## React vs React Native

| Özellik       | React                   | React Native                       |
|---------------|-------------------------|------------------------------------|
| **Platform**  | Web tarayıcıları        | iOS ve Android mobil uygulamalar   |
| **Çıktı**     | HTML, CSS               | Native mobil bileşenler            |
| **Bileşenler**| `<div>`, `<span>`, `<p>`| `<View>`, `<Text>`, `<Image>`      |
| **Styling**   | CSS, Sass, Tailwind     | StyleSheet (CSS benzeri JS objesi) |
| **Amaç**      | Web sitesi/uygulaması   | Mobil uygulama                     |

**Ortak Noktalar:**
- Aynı component mantığı
- JSX söz dizimi
- React Hooks kullanımı
- State ve props yönetimi

> React öğrenirseniz, React Native'e geçiş çok kolay olur!

---

## Avantajları

- **Kolay Öğrenme Eğrisi**: JavaScript biliyorsanız hızlıca başlayabilirsiniz
- **Yeniden Kullanılabilir Componentler**: Kod tekrarını azaltır
- **Virtual DOM**: Yüksek performans
- **JSX**: HTML benzeri söz dizimi ile sezgisel geliştirme
- **Tek Yönlü Veri Akışı**: Daha öngörülebilir kod
- **Developer Tools**: Güçlü debugging araçları
- **SEO Dostu**: Server-side rendering desteği (Next.js ile)
- **Geniş Ekosistem**: Redux, React Router, Material UI vb.

---

## Dezavantajları

- **Sadece UI Kütüphanesi**: Tam bir framework değil, ek araçlar gerektirir
- **JSX Öğrenme**: Yeni başlayanlar için kafa karıştırıcı olabilir
- **Hızlı Değişim**: Sürekli güncellenen ekosistem takip gerektirir
- **Karar Yorgunluğu**: Çok fazla seçenek (state management, routing vb.)
- **Bundle Boyutu**: Küçük projeler için fazla olabilir
- **SEO Zorlukları**: Client-side rendering varsayılan olarak SEO dostu değil

## Bilinmesi Gereken Kilit Yapılar

### 1. JSX (JavaScript XML)
HTML benzeri söz dizimi ile JavaScript içinde UI yazmanızı sağlar.

```jsx
const element = <h1>Merhaba, React!</h1>;
```

### 2. Components (Bileşenler)
UI'ın yapı taşlarıdır. İki türü vardır:

```jsx
// Fonksiyonel Component (Modern yaklaşım)
function Greeting({ name }) {
  return <h1>Merhaba, {name}!</h1>;
}

// Class Component (Eski yaklaşım)
class Greeting extends React.Component {
  render() {
    return <h1>Merhaba, {this.props.name}!</h1>;
  }
}
```

### 3. Props (Properties)
Component'lere dışarıdan veri aktarmak için kullanılır.

```jsx
<UserCard name="Ahmet" age={25} />
```

### 4. State
Component'in kendi içindeki değişebilir verisidir.

```jsx
const [count, setCount] = useState(0);
```

### 5. Hooks
Fonksiyonel component'lerde state ve lifecycle kullanımını sağlar.

| Hook         | Kullanım Amacı |
|--------------|----------------|
| `useState`   | State yönetimi |
| `useEffect`  | Yan etkiler (API çağrıları, DOM manipülasyonu) |
| `useContext` | Context API ile global state |
| `useRef`     | DOM referansları ve değişken saklama |
| `useMemo`    | Hesaplama sonuçlarını önbellekleme |
| `useCallback`| Fonksiyonları önbellekleme |

### 6. Virtual DOM
React, gerçek DOM'un hafif bir kopyasını tutar. Değişiklikler önce Virtual DOM'da yapılır, sonra en verimli şekilde gerçek DOM'a aktarılır.

```
State Değişikliği → Virtual DOM Güncellenir → Diff Algoritması →
Sadece Değişen Kısımlar Gerçek DOM'a Yazılır
```

### 7. Lifecycle (Yaşam Döngüsü)
Component'in mount, update ve unmount aşamalarını yönetir.

```jsx
useEffect(() => {
  // Component mount olduğunda çalışır
  console.log('Component yüklendi');

  return () => {
    // Component unmount olduğunda çalışır (cleanup)
    console.log('Component kaldırıldı');
  };
}, []);
```

## Özet

React, modern web geliştirmenin temel taşlarından biridir. Component tabanlı mimarisi, Virtual DOM performansı ve güçlü ekosistemi ile hem küçük hem de büyük ölçekli projeler için ideal bir seçimdir.

Sonraki derste **JSX söz dizimi** ve **ilk component**'imizi oluşturacağız!
